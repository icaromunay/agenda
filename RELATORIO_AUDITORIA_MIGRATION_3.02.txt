AUDITORIA DE MIGRATION — MUNAY AGENDA PRO 3.02

1. OBJETIVO
Validar e corrigir a migration 001_init.sql para compatibilidade total com PostgreSQL/Neon, eliminando o erro 42P17 (functions in index expression must be marked IMMUTABLE).

2. CAUSA RAIZ IDENTIFICADA
O índice problemático era o índice funcional de clientes por nome + WhatsApp.
Em versões anteriores, ele utilizava função não-imutável em expressão de índice (ex.: unaccent(name) combinado com lower(...), e possivelmente normalização adicional).
No PostgreSQL/Neon, funções usadas em CREATE INDEX expression precisam ser IMMUTABLE.

3. CORREÇÃO APLICADA NA 3.02
O índice foi simplificado para uma expressão compatível com Neon:
CREATE INDEX IF NOT EXISTS idx_clients_name_whatsapp ON clients((lower(name)), whatsapp);

4. RESULTADO DA AUDITORIA DO ARQUIVO migrations/001_init.sql
- Extensões: pgcrypto e unaccent carregadas sem problema.
- Tabelas: sem incompatibilidades com Neon.
- ALTER TABLE IF NOT EXISTS: compatíveis.
- Índices comuns: compatíveis.
- Índice funcional revisado: agora compatível com Neon/PostgreSQL.

5. VALIDAÇÃO EXECUTADA
Foi criado um banco temporário vazio no próprio PostgreSQL remoto e executado:
- npm run migrate
Resultado:
- Migrações executadas com sucesso
- Tabelas criadas com sucesso
- Índices criados com sucesso
